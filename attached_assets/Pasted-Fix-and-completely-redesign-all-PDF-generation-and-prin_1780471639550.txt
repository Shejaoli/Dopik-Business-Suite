## PART 1 — COMPLETE PDF & PRINT REDESIGN

Every document that can be printed or downloaded as PDF must be completely redesigned to look professional, clean, and beautiful. Apply this to ALL of the following: Sales Receipt, Purchase Order, Repair Invoice, Expense Report, Sales Report, Purchase Report, Summary Report, Profitability Report, Stock Audit Report, Credit Statement.

**Global PDF Design Standards (apply to every document):**

**Page Setup:**
- A4 paper size (210mm × 297mm)
- Margins: 15mm top, 15mm bottom, 20mm left, 20mm right
- Clean white background
- Use a professional font: Inter or Roboto (import from Google Fonts)
- Font sizes: Header 22px bold, Section titles 13px bold, Body text 11px, Small print 9px

**Header (every document):**
```
┌─────────────────────────────────────────────────────┐
│  [D logo circle - Dopik blue #1A6DB5]               │
│                                                      │
│         DOPIK ELECTRONICS                           │
│         Kigali, Rwanda                              │
│         +250 788 000 000 | info@dopikelectronics.com│
│         www.dopikelectronics.com                    │
│                                                      │
│  ════════════════════════════════════════════════   │
│  [DOCUMENT TITLE IN CAPS]        [Doc Number]       │
│                                  [Date]             │
└─────────────────────────────────────────────────────┘
```
- Company name in Dopik blue (#1A6DB5), bold, large
- Full blue horizontal divider line below company info
- Document title on the left, document number and date on the right

**Tables (for all documents with item tables):**
- Header row: Dopik blue (#1A6DB5) background, white text, bold
- Alternating row colors: white and very light gray (#F8F9FA)
- Clean thin borders (#E0E0E0)
- Numbers right-aligned in their columns
- Text left-aligned
- Column widths properly proportioned — no truncated text ever

**Footer (every document):**
```
════════════════════════════════════════════════════
Generated by Dopik Electronics Management System
[Date and time generated] | Confidential — Internal Use
www.dopikelectronics.com | +250 788 000 000
                                          Page 1 of 1
```
- Thin blue line above footer
- Page number bottom right
- Generation timestamp bottom left

**Print CSS:**
- When printing, hide completely: sidebar navigation, top navigation bar, all buttons, browser URL bar artifacts, any chat widgets
- Show ONLY the document content
- Use `@media print` CSS with `display: none` on all UI elements outside the print area
- Ensure colors print correctly: add `-webkit-print-color-adjust: exact` and `print-color-adjust: exact`
- No page breaks in the middle of table rows
- Page break before footer if content is long

## PART 5 — ALL REPORTS: DATE RANGE & PERIOD FILTERING

Add comprehensive date filtering to ALL report pages: Sales Report, Purchase Report, Expense Report, Summary Report, Profitability Report, Credit Report, Stock Report.

**Filter bar at the top of every report page:**

```
┌──────────────────────────────────────────────────────────────┐
│  Period: [This Week] [This Month] [This Year] [Custom Range] │
│                                                              │
│  Custom: From [____/____/____] To [____/____/____]          │
│                                                              │
│  [Generate Report]  [Export PDF]  [Export Excel/CSV]        │
└──────────────────────────────────────────────────────────────┘
```

**Period buttons behavior:**
- **This Week** — Monday to today of the current week
- **This Month** — 1st of current month to today
- **This Year** — 1st January of current year to today
- **Custom Range** — shows two date picker fields (From and To). To date cannot be before From date
- Default on page load: This Month
- Selected period button is highlighted in Dopik blue

**Each report must show at the top:**
```
SALES REPORT — June 2026
Period: 01 June 2026 to 02 June 2026
Generated: 02 June 2026 at 14:30
```

**Weekly Report view:**
When "This Week" is selected, group data by day of the week:
- Monday: [sales count] — [total RWF]
- Tuesday: [sales count] — [total RWF]
- etc.
- Show a bar chart of daily totals for the week

**Monthly Report view:**
When "This Month" is selected, group data by week:
- Week 1 (1–7 June): [totals]
- Week 2 (8–14 June): [totals]
- etc.
- Show a line chart of daily totals for the month

**Annual Report view:**
When "This Year" is selected, group data by month:
- January: [totals]
- February: [totals]
- etc.
- Show a bar chart of monthly totals for the year

**Sales Report sections:**
- Summary cards: Total Sales, Total Revenue, Total Items Sold, Average Sale Value
- Sales by payment method breakdown
- Top 5 customers by revenue
- Top 5 products by revenue
- Detailed sales table with all transactions in the period

**Purchase Report sections:**
- Summary cards: Total Purchases, Total Spent, Total Units Received, Average Purchase Value
- Spending by supplier breakdown
- Spending by category (Phones/Computers/Accessories)
- Detailed purchases table

**Expense Report sections:**
- Summary cards: Total Expenses, by category breakdown
- Budget vs Actual chart per category
- Detailed expense table

**Summary Report (Profit & Loss) sections:**
- Revenue: total sales revenue in period
- Cost of Goods Sold: total purchase cost of items sold in period
- Gross Profit: Revenue − COGS
- Operating Expenses: total expenses in period
- Net Profit: Gross Profit − Expenses
- Profit Margin %
- Comparison with previous same period: *"vs last month: +12%"*
- Month-by-month trend chart

**PDF Export for Reports:**
Every report export must use the global PDF design standards defined in Part 1. The exported PDF must exactly match what is shown on screen — same data, same grouping, same charts (render charts as images in the PDF).

**Excel/CSV Export:**
- Excel export uses SheetJS with proper formatting: bold headers, number formatting with commas, date formatting
- Multiple sheets in one Excel file: Summary sheet + Detail sheet
- CSV export is plain data for accountants and RRA compliance

**API endpoints to update:**
All report endpoints must accept `?from=DATE&to=DATE` query parameters:
- `GET /api/reports/sales?from=2026-06-01&to=2026-06-30`

## PART 6 — PDF GENERATION LIBRARY

Use a proper PDF generation approach:

**Option A (recommended):** Use `@react-pdf/renderer` on the frontend to generate PDFs entirely in React with proper styling, fonts, and layout control.

**Option B:** Use `puppeteer` on the server to render a styled HTML page and convert to PDF.

**Choose whichever is already installed or easier to implement. Either way:**
- PDF must render exactly as designed in Parts 2–5 above
- Fonts must embed correctly (no font substitution)
- Colors must render correctly (Dopik blue must appear blue, not gray)
- Tables must not break across pages mid-row
- Images (logo) must render in the PDF
- File must download with a meaningful name: e.g. `Dopik-Receipt-REC-20260602-0008.pdf`, `Dopik-Sales-Report-June-2026.pdf`
