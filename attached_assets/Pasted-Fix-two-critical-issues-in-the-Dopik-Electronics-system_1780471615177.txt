## FIX 2 — ITEM DUPLICATE DETECTION (SMART ITEM SUGGESTION)

When adding a new product/item anywhere in the system, detect if a similar item already exists and alert the admin before creating a duplicate.

**Where this applies:**
- Add New Product form (Items/Products page)
- Add Item inline (if available in purchase form)

**How it works:**

**Real-time duplicate detection while typing:**
- As the admin types the product name in the Name field, after 3 or more characters, automatically search the existing products database for matches
- Search must be: case-insensitive, partial match, and also catch common variations e.g. "iPhone16" matches "iPhone 16", "iphone 16", "Apple iPhone 16"
- Show a dropdown suggestion panel below the name field showing matching existing products:

```
⚠️ Similar items already exist:

📱 Apple iPhone 16 — 44 units in stock — [View] [Use This Instead]
📱 iPhone 16 Pro — 12 units in stock — [View] [Use This Instead]

[ Continue creating new item anyway ]
```

**Suggestion panel behavior:**
- **View** button — opens the existing product in a new tab so admin can check if it is the same item
- **Use This Instead** button — closes the Add New Product form and navigates to the existing product's edit/stock page instead
- **Continue creating new item anyway** link — dismisses the suggestions and lets admin proceed with creating a new product
- If no similar items found — no suggestion panel shown, form works normally

**Matching logic:**
- Strip spaces, hyphens, and capitalize before comparing
- Match on: exact name, name contains search term, search term contains name
- Also match on brand + model combination if those fields exist
- Examples that should match:
  - "iPhone 16" → matches "Apple iPhone 16", "iphone16", "iPhone16 Pro" (partial)
  - "Samsung A55" → matches "Samsung Galaxy A55", "Galaxy A55"
  - "AirPods Pro" → matches "Apple AirPods Pro 3", "Airpods pro"

**In Purchase Form — item card dropdown:**
- When admin selects an item from the dropdown in a purchase card, if they have already added the same product in another card in the same purchase, show a warning badge on the card:
  - *"⚠️ You already have [Product Name] in another row of this purchase. Make sure these are different units."*
  - This is a warning only — not a blocker, since buying multiple units of the same product is valid

**In Sale Form — item card dropdown:**
- Same warning if the same product is added twice in one sale
- For serialized items this is already handled (same IMEI can't be selected twice) but for non-serialized accessories, show: *"ℹ️ You already have [Product] in this sale. Consider increasing the quantity instead."*
- Show a **"Merge"** button that combines the two rows into one with combined quantity

**API endpoint for duplicate check:**
- `GET /api/products/search?name=[query]` — returns matching products with name, category, stock count
- Must respond within 300ms for good UX — add database index on product name column if not already present
- Rate limit: maximum 1 request per 500ms (debounce on frontend)

**Apply all fixes without breaking any existing product, sale, customer, or purchase data.**
