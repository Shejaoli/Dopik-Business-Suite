DATABASE SCHEMA
Create the following PostgreSQL tables. All monetary values stored as NUMERIC(12,2). All timestamps as TIMESTAMP DEFAULT NOW().
sql-- Users (auth)
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  email VARCHAR(150) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  role VARCHAR(20) DEFAULT 'user',   -- 'admin' or 'user'
  status VARCHAR(20) DEFAULT 'active',
  created_at TIMESTAMP DEFAULT NOW()
);

-- Items (product catalog)
CREATE TABLE items (
  id SERIAL PRIMARY KEY,
  name VARCHAR(200) NOT NULL,
  qty_type VARCHAR(50) NOT NULL,     -- 'piece', 'kg', 'box', 'M', 'pack', etc.
  purchase_price NUMERIC(12,2) NOT NULL DEFAULT 0,
  sale_price NUMERIC(12,2) NOT NULL DEFAULT 0,
  alternative_item_id INT REFERENCES items(id),
  created_at TIMESTAMP DEFAULT NOW()
);

-- Stock
CREATE TABLE stock (
  id SERIAL PRIMARY KEY,
  item_id INT NOT NULL REFERENCES items(id) ON DELETE CASCADE,
  quantity NUMERIC(12,2) NOT NULL DEFAULT 0,
  min_stock NUMERIC(12,2) NOT NULL DEFAULT 0,
  updated_at TIMESTAMP DEFAULT NOW()
);

-- Stock Adjustments
CREATE TABLE stock_adjustments (
  id SERIAL PRIMARY KEY,
  item_id INT NOT NULL REFERENCES items(id),
  adjustment_type VARCHAR(10) NOT NULL,  -- 'increase' or 'decrease'
  quantity NUMERIC(12,2) NOT NULL,
  previous_qty NUMERIC(12,2) NOT NULL,
  new_qty NUMERIC(12,2) NOT NULL,
  reason TEXT,
  adjusted_by INT REFERENCES users(id),
  created_at TIMESTAMP DEFAULT NOW()
);

-- Vendors
CREATE TABLE vendors (
  id SERIAL PRIMARY KEY,
  name VARCHAR(150) NOT NULL,
  contact_person VARCHAR(100),
  email VARCHAR(150),
  phone VARCHAR(30),
  address TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Customers
CREATE TABLE customers (
  id SERIAL PRIMARY KEY,
  name VARCHAR(150) NOT NULL,
  contact_person VARCHAR(100),
  email VARCHAR(150),
  phone VARCHAR(30),
  address TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Purchases
CREATE TABLE purchases (
  id SERIAL PRIMARY KEY,
  item_id INT NOT NULL REFERENCES items(id),
  quantity NUMERIC(12,2) NOT NULL,
  total_cost NUMERIC(12,2) NOT NULL,
  vendor_id INT REFERENCES vendors(id),
  payment_method VARCHAR(30) DEFAULT 'cash',  -- 'cash', 'bank', 'mobile_money', 'credit'
  recorded_by INT REFERENCES users(id),
  created_at TIMESTAMP DEFAULT NOW()
);

-- Sales
CREATE TABLE sales (
  id SERIAL PRIMARY KEY,
  customer_id INT REFERENCES customers(id),
  payment_method VARCHAR(30) DEFAULT 'cash',  -- 'cash', 'bank', 'mobile_money', 'credit'
  total_amount NUMERIC(12,2) NOT NULL DEFAULT 0,
  recorded_by INT REFERENCES users(id),
  created_at TIMESTAMP DEFAULT NOW()
);

-- Sale Items (for multi-item sales)
CREATE TABLE sale_items (
  id SERIAL PRIMARY KEY,
  sale_id INT NOT NULL REFERENCES sales(id) ON DELETE CASCADE,
  item_id INT NOT NULL REFERENCES items(id),
  quantity NUMERIC(12,2) NOT NULL,
  unit_price NUMERIC(12,2) NOT NULL,
  line_total NUMERIC(12,2) NOT NULL
);

-- Receivables (credit sales tracking)
CREATE TABLE receivables (
  id SERIAL PRIMARY KEY,
  sale_id INT NOT NULL REFERENCES sales(id),
  customer_id INT NOT NULL REFERENCES customers(id),
  total_amount NUMERIC(12,2) NOT NULL,
  paid_amount NUMERIC(12,2) NOT NULL DEFAULT 0,
  due_date DATE,
  status VARCHAR(20) DEFAULT 'unpaid',   -- 'unpaid', 'partial', 'paid'
  created_at TIMESTAMP DEFAULT NOW()
);

-- Receivable Payments
CREATE TABLE receivable_payments (
  id SERIAL PRIMARY KEY,
  receivable_id INT NOT NULL REFERENCES receivables(id),
  amount NUMERIC(12,2) NOT NULL,
  payment_method VARCHAR(30),
  paid_at TIMESTAMP DEFAULT NOW(),
  recorded_by INT REFERENCES users(id)
);

-- Payables (credit purchases tracking)
CREATE TABLE payables (
  id SERIAL PRIMARY KEY,
  purchase_id INT NOT NULL REFERENCES purchases(id),
  vendor_id INT REFERENCES vendors(id),
  total_amount NUMERIC(12,2) NOT NULL,
  paid_amount NUMERIC(12,2) NOT NULL DEFAULT 0,
  due_date DATE,
  status VARCHAR(20) DEFAULT 'unpaid',
  created_at TIMESTAMP DEFAULT NOW()
);

-- Expense Accounts
CREATE TABLE expense_accounts (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  account_type VARCHAR(30) DEFAULT 'expense',
  created_at TIMESTAMP DEFAULT NOW()
);

-- Expenses
CREATE TABLE expenses (
  id SERIAL PRIMARY KEY,
  account_id INT NOT NULL REFERENCES expense_accounts(id),
  payment_method VARCHAR(30) NOT NULL,  -- 'cash', 'bank', 'mobile_money'
  amount NUMERIC(12,2) NOT NULL,
  description TEXT,
  paid_by INT REFERENCES users(id),
  created_at TIMESTAMP DEFAULT NOW()
);

-- Cash/Bank/Mobile Money Balances (ledger)
CREATE TABLE balances (
  id SERIAL PRIMARY KEY,
  method VARCHAR(30) UNIQUE NOT NULL,  -- 'cash', 'bank', 'mobile_money'
  amount NUMERIC(12,2) NOT NULL DEFAULT 0,
  updated_at TIMESTAMP DEFAULT NOW()
);

-- Audit Log
CREATE TABLE audit_logs (
  id SERIAL PRIMARY KEY,
  user_id INT REFERENCES users(id),
  action VARCHAR(200),
  details TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

BALANCE LEDGER LOGIC (CRITICAL — HOW MONEY FLOWS)
The system tracks three payment method balances: Cash, Bank, Mobile Money. These balances update automatically on every transaction:

New Sale (Cash/Bank/Mobile Money): → Add sale total to that balance
New Sale (Credit): → Create a Receivable. Balance only updates when payment is recorded on the receivable.
New Purchase (Cash/Bank/Mobile Money): → Subtract purchase total from that balance
New Purchase (Credit): → Create a Payable. Balance only updates when payment is recorded on the payable.
Pay Expense: → Subtract expense amount from chosen balance
Record Receivable Payment: → Add paid amount to chosen balance, update receivable paid_amount and status
Record Payable Payment: → Subtract paid amount from chosen balance, update payable status

Initialize balances table with:
sqlINSERT INTO balances (method, amount) VALUES ('cash', 0), ('bank', 0), ('mobile_money', 0);

STOCK LOGIC (CRITICAL — HOW STOCK FLOWS)
Stock updates automatically on every relevant transaction:

New Purchase recorded: → stock.quantity += quantity_purchased for that item
New Sale recorded: → stock.quantity -= quantity_sold for that item (block sale if stock = 0)
Sale Reverted: → stock.quantity += quantity_sold back to stock
Stock Adjustment (increase): → stock.quantity += adjustment_quantity
Stock Adjustment (decrease): → stock.quantity -= adjustment_quantity
Stock status logic:

quantity = 0 → OUT OF STOCK
0 < quantity <= min_stock → LOW STOCK
quantity > min_stock → IN STOCK (normal)


When adding a new item, automatically create a stock row for it with quantity = 0 and min_stock = 0.
Out-of-stock items are disabled and cannot be selected in the New Sale form. Show them grayed out with a tooltip "Out of stock".


GROSS PROFIT LOGIC (FOR INCOME STATEMENT)
Gross Profit per sale item = (unit_price - item.purchase_price) × quantity_sold
Monthly Gross Profit = SUM of all sale item gross profits in that month
Net Profit = Gross Profit - Stock Adjustment Cost - Total Expenses
Stock Adjustment Cost = SUM of (decrease adjustments × item.purchase_price) in that month

INTER-MODULE CONNECTIONS (HOW EVERYTHING LINKS)

Items → Stock: Every item has exactly one stock row. Creating an item auto-creates stock. Deleting an item cascades to stock.
Items → Sale/Purchase forms: All sale/purchase forms use a searchable item dropdown populated from the items table. Out-of-stock items are disabled in sale forms.
Purchases → Stock → Balance: Recording a purchase increases that item's stock quantity AND decreases the chosen payment method balance (or creates a payable if credit).
Sales → Stock → Balance: Recording a sale decreases stock AND increases the chosen payment method balance (or creates a receivable if credit).
Sales Revert → Stock → Balance: Reverting a sale restores stock and reverses the balance change.
Stock Adjustment → Stock → Audit Log: Every adjustment is logged with who did it, when, and why.
Stock Alerts → Stock Adjustment / Purchase: The Restock button on stock alerts pre-fills the New Purchase form with that item.
Receivables → Balance: Recording payment on a receivable adds money to the chosen balance.
Payables → Balance: Recording payment on a payable deducts money from the chosen balance.
Pay Expense → Balance: Expenses deduct from the chosen balance. Cannot exceed available balance.
Income Statement: Pulls gross profit from sales + sale_items (unit_price - purchase_price × qty), expenses from expenses table, and stock adjustment cost from decrease adjustments × purchase price. All computed server-side per month.
Item History: Aggregates data from purchases, sale_items, and stock_adjustments all filtered by item_id, sorted by date descending.
Customers → Sales → Receivables: Selecting a customer on a credit sale auto-links that receivable to the customer.
Vendors → Purchases → Payables: Selecting a vendor on a credit purchase auto-links that payable to the vendor.
Audit Log: Every data-changing operation (sale, purchase, adjustment, expense, payment) writes to audit_logs with user_id, action name, and relevant details.
