**Period buttons — exact behavior:**

- **Today** — from 00:00 to 23:59 of today only. Shows hourly breakdown
- **Yesterday** — from 00:00 to 23:59 of yesterday. Shows hourly breakdown

- **This Month** — 1st of current month 00:00 to today 23:59. Shows weekly breakdown (Week 1, Week 2...)
- **This Year** — 1st January 00:00 to today 23:59. Shows monthly breakdown (Jan, Feb, Mar...)
- **Custom Range** — admin picks exact From and To dates using date pickers. To date cannot be before From date. Shows daily breakdown if range ≤ 31 days, weekly if ≤ 90 days, monthly if longer

## PART 2 — SMART BACKUP SYSTEM WITH GOOGLE DRIVE & ONEDRIVE

Implement a smart backup trigger system that prompts the admin to back up data at key moments and automatically syncs to their connected cloud storage.

**SECTION A — Cloud Storage Connection Setup**

In Settings, add a **"Cloud Backup"** section with two connection options:

**Google Drive connection:**
- Button: **"Connect Google Drive"**
- Uses Google OAuth2 — clicking opens Google's OAuth consent screen
- Admin selects their Google account and grants Drive access
- After successful connection: show connected account email, green checkmark, and "Disconnect" button
- Store OAuth tokens securely in the database (encrypted)
- Backup folder created automatically: `Dopik Electronics Backups` in the root of their Google Drive

**Microsoft OneDrive connection:**
- Button: **"Connect OneDrive"**
- Uses Microsoft OAuth2 — clicking opens Microsoft login
- Same flow as Google Drive
- Backup folder: `Dopik Electronics Backups` in OneDrive root

**Connection status display:**
```
┌──────────────────────────────────────────────────┐
│  ☁️ Cloud Backup                                  │
│                                                  │
│  Google Drive: ✅ Connected (sheja@gmail.com)    │
│  Last backup: 02 June 2026 at 10:45              │
│  [Backup Now] [Disconnect]                       │
│                                                  │
│  OneDrive: ⬜ Not connected                      │
│  [Connect OneDrive]                              │
│                                                  │
│  Automatic backup: ✅ Enabled                    │
│  Backup frequency: [Every login / Hourly / Daily]│
└──────────────────────────────────────────────────┘
```

Admin can connect both simultaneously — backup goes to both.

**SECTION B — Login Backup Prompt**

When any admin or staff logs in successfully, immediately show a backup prompt popup:

```
┌──────────────────────────────────────────────────┐
│  ☁️ Cloud Backup                                  │
│                                                  │
│  Last backup: 2 days ago (31 May 2026)           │
│  New data since last backup:                     │
│  • 8 new sales (3,200,000 RWF)                   │
│  • 3 new purchases                               │
│  • 2 new customers                               │
│  • 1 repair completed                            │
│                                                  │
│  Backup destination: Google Drive ✅             │
│                                                  │
│  [⏭ Skip for now]  [☁️ Backup Now]              │
│                                                  │
│  □ Don't ask me again today                      │
└──────────────────────────────────────────────────┘
```

**Popup behavior:**
- Only shows if last backup was more than 6 hours ago
- If last backup was within 6 hours: skip the popup silently
- **"Backup Now"** button triggers immediate backup upload to connected cloud storage
- **"Skip for now"** dismisses the popup — will show again next login
- **"Don't ask me again today"** — dismisses and suppresses for the rest of the calendar day (resets at midnight)
- If no cloud storage is connected: show a different message:
```
│  ⚠️ No cloud backup configured                   │
│  Connect Google Drive or OneDrive to enable      │
│  automatic backups and protect your data.        │
│                                                  │
│  [⏭ Skip]  [⚙️ Set Up Backup]                   │
```
- "Set Up Backup" navigates to Settings > Cloud Backup section

**SECTION C — Logout Backup Prompt**

When admin clicks Log Out, before logging out show:

```
┌──────────────────────────────────────────────────┐
│  Before you go...                                │
│                                                  │
│  Last backup: 3 hours ago                        │
│  [X] Back up to Google Drive before logging out  │
│                                                  │
│  [Log Out Without Backup]  [Backup & Log Out]    │
└──────────────────────────────────────────────────┘
```

- Checkbox is checked by default if last backup > 2 hours ago
- Checkbox unchecked by default if last backup < 2 hours ago
- **"Backup & Log Out"** — runs backup, shows progress bar, then logs out when complete
- **"Log Out Without Backup"** — logs out immediately
- If no cloud storage connected: skip this popup entirely and log out normally

**SECTION D — Automatic Scheduled Backup**

In Settings, admin can configure automatic backup frequency:
- Every login (triggers on each successful login — separate from the popup, runs silently in background)
- Every 6 hours (server-side cron)
- Every 12 hours (server-side cron)
- Daily at a set time (server-side cron, time picker)
- Weekly (every Monday at set time)

Silent background backup — no popup, just updates the "Last backup" timestamp and logs it in the activity log.

**SECTION E — What Gets Backed Up**

Each backup is a compressed JSON file containing all business data:
- All sales and sale items
- All purchases and purchase items
- All serialized units (with IMEI records)
- All customers
- All credit accounts and payments
- All expenses
- All repair jobs
- All products and categories
- All vendors
- All receipts metadata
- Settings and configuration

Backup file naming: `Dopik-Backup-[ShopName]-[YYYY-MM-DD-HH-MM].json.gz`

- Last 30 backups shown
- Failed backups show error reason on hover
- **"Download"** button on each row to re-download that backup locally
- **"Restore from this backup"** button (with the existing 2-step safety confirmation)

**SECTION G — Backup Progress UI**

When a backup is running (triggered manually or at login/logout):
- Show a non-blocking progress bar at the bottom of the screen (like a toast):
```
☁️ Backing up to Google Drive... ████████░░ 80%  [Cancel]
```
- On success: *"✅ Backup complete — saved to Google Drive (2.3 MB)"* — dismisses after 5 seconds
- On failure: *"❌ Backup failed: [error reason]. [Retry] [Dismiss]"* — stays until dismissed
